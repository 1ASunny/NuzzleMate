# 目标拆解与需求优先级分析

**项目：基于大模型的智能宠物健康管理与辅助诊疗系统**  
**日期：2026-05-18**

---

## 1. 目标拆解

本系统的目标不能只理解为“做一个 AI 宠物医生”，而应拆解为三层：

1. 用户目标：让宠主在宠物健康问题上更安心、更高效、更少误判。
2. 业务目标：通过 AI 分诊、健康管理、兽医服务、医院导流和商品履约形成商业闭环。
3. 数据目标：沉淀宠物健康档案、问诊行为、症状数据、医生反馈和服务转化数据，持续优化产品和模型能力。

---

## 2. 用户目标

### 2.1 降低宠主健康判断成本

用户最核心目标是：宠物出现异常时，能够快速知道“现在严不严重、该不该去医院、下一步该怎么做”。

系统需要帮助用户完成：快速描述症状、上传图片/语音/视频、通过 AI 多轮问诊补齐关键信息、获得风险等级判断、明确观察/线上问诊/线下就医、获得就医前准备清单。

### 2.2 建立长期宠物健康档案

| 类型 | 内容 |
|---|---|
| 基础档案 | 宠物姓名、物种、品种、年龄、性别、体重、是否绝育 |
| 医疗档案 | 疫苗、驱虫、过敏史、既往病史、手术史 |
| 日常记录 | 饮食、饮水、排便、体重、精神状态 |
| 用药记录 | 药名、频次、剂量来源、用药时间、不良反应 |
| 问诊记录 | AI 问诊摘要、医生建议、线下就医记录 |
| 检查资料 | 化验单、影像报告、处方、病历照片 |

### 2.3 提高日常健康管理效率

系统帮助用户完成定时喂食、饮水记录、用药提醒、驱虫提醒、疫苗提醒、复诊提醒、体重记录、术后护理提醒、慢病管理提醒。

### 2.4 降低宠主焦虑感

系统需要帮助用户快速判断是否急症、避免被碎片化搜索吓到、避免被社区经验误导、明确哪些症状必须马上就医、必要时转接线上兽医或推荐线下医院。

### 2.5 提升就医效率

系统自动生成就医摘要，包括宠物基础信息、当前症状、症状持续时间、精神食欲变化、排便排尿情况、既往病史、当前用药、上传图片和视频、AI 风险分级、用户最关心的问题。

## 3. 业务目标

### 3.1 短期目标：验证核心需求和产品闭环

短期核心业务目标：验证 AI 分诊需求、验证宠物档案价值、验证提醒功能留存、验证兽医转化、验证医院导流、验证商业转化。

最小闭环：

```text
AI免费初筛
↓
建立宠物档案
↓
生成风险分级
↓
转接兽医或推荐医院
↓
沉淀问诊记录
↓
后续提醒和复诊管理
```

### 3.2 中期目标：形成健康管理留存和多元收入

| 方向 | 目标 |
|---|---|
| 用户留存 | 通过提醒、档案、健康报告提高月活和复访 |
| 会员转化 | 推出基础会员、高级会员、多宠会员、慢病管理会员 |
| 医生服务 | 扩大线上兽医供给，提高接诊效率和满意度 |
| 医院合作 | 接入更多线下宠物医院，形成转诊和复诊闭环 |
| 商品履约 | 接入药品、处方粮、护理用品、主粮、智能硬件 |
| 医生工作台 | 为兽医提供病情摘要、病例管理、复诊管理工具 |

### 3.3 长期目标：成为宠物健康管理入口

长期产品形态：

```text
C端宠主健康管理平台
+
B端兽医工作台
+
医院数字化协作平台
+
药品用品履约平台
+
智能硬件健康数据中台
```

## 4. 数据目标

### 4.1 用户增长数据目标

| 指标 | 含义 |
|---|---|
| 注册用户数 | 有多少用户进入平台 |
| 新增用户数 | 每日/每周/每月新增用户 |
| 宠物档案创建数 | 用户是否愿意录入宠物信息 |
| 宠物档案完成率 | 档案信息是否足够完整 |
| 多宠用户比例 | 是否吸引多宠家庭 |
| 首次问诊转化率 | 新用户是否进入 AI 问诊 |
| 新手引导完成率 | 用户是否完成初始档案、提醒设置 |

### 4.2 AI 问诊数据目标

| 指标 | 含义 |
|---|---|
| AI 问诊次数 | 用户使用 AI 问诊的频率 |
| 问诊完成率 | 用户是否完成多轮问诊 |
| 平均问诊轮次 | 问诊是否过长或过短 |
| 图片上传率 | 用户是否愿意使用多模态能力 |
| 语音输入率 | 语音功能是否有价值 |
| 风险分级分布 | 低/中/高/急症风险占比 |
| 高风险转人工率 | 高风险用户是否被转接医生 |
| 急症提示触发率 | 急症规则是否有效工作 |
| AI 回答满意度 | 用户对 AI 建议是否认可 |

### 4.3 医疗安全数据目标

| 指标 | 目标意义 |
|---|---|
| 急症识别召回率 | 真正危险情况是否被识别出来 |
| 急症漏报率 | 危险情况被误判为低风险的比例 |
| 高风险转医生率 | 高风险情况是否进入人工承接 |
| 高风险转医院率 | 是否有效引导线下就医 |
| 医生一致率 | AI 风险分级和兽医判断是否一致 |
| 危险建议率 | AI 是否给出错误用药、延误就医等危险建议 |
| 人工抽检通过率 | AI 回答经过人工审核后的合格比例 |
| 用户投诉率 | 医疗建议是否引发用户投诉 |

### 4.4 商业转化数据目标

| 指标 | 含义 |
|---|---|
| 线上问诊付费转化率 | AI 问诊后是否愿意付费问兽医 |
| 问诊订单数 | 实际付费问诊规模 |
| 问诊客单价 | 每次问诊平均收入 |
| 医生接诊完成率 | 医生服务是否稳定 |
| 医院预约转化率 | 高风险用户是否转线下 |
| 商品点击率 | 药品用品推荐是否有效 |
| 商品购买转化率 | 是否形成电商收入 |
| 会员转化率 | 用户是否愿意订阅 |
| 会员续费率 | 会员价值是否持续 |
| ARPU | 平均每用户收入 |
| LTV | 用户生命周期价值 |
| CAC | 获客成本 |
| LTV/CAC | 单位获客投入是否划算 |

## 5. MoSCoW 需求优先级分析

### 5.1 划分原则

| 判断维度 | 说明 |
|---|---|
| 用户刚需程度 | 是否直接解决用户最核心的健康焦虑 |
| 医疗安全价值 | 是否能降低误判、漏判和错误用药风险 |
| 商业闭环价值 | 是否能带来问诊、医院、会员、商品等转化 |
| 开发复杂度 | 是否适合在 MVP 阶段快速上线验证 |

---

### 5.2 Must Have：必须有

| 功能模块 | 为什么必须有 |
|---|---|
| 宠物健康档案 | AI 判断和长期管理基础 |
| AI 图文语音问诊 | 核心入口 |
| 多轮问诊 | 补齐关键信息 |
| 急症红旗识别 | 医疗安全底线 |
| 风险分级 | 帮用户判断下一步 |
| 病情摘要生成 | 连接医生和医院 |
| 线上兽医转接 | 合规和信任承接 |
| 附近医院推荐 | 高风险线下闭环 |
| 用药提醒 | 问诊后管理 |
| 喂食、疫苗、驱虫提醒 | 提高日常留存 |
| 基础科普和急救科普 | 降低错误认知 |

---

### 5.3 Should Have：应该有

| 功能模块 | 价值 |
|---|---|
| 医生工作台 | 提高兽医效率 |
| 检查报告识别 | 支持复诊和慢病 |
| 复诊计划管理 | 提升闭环能力 |
| 视频上传识别 | 增强多模态能力 |
| 药品用品电商接入 | 商业化增强 |
| 会员体系 | 中期收入来源 |
| 慢病管理模块 | 高价值用户服务 |
| 医院预约转诊 | 线下闭环增强 |

---

### 5.4 Could Have：可以有

| 功能模块 | 说明 |
|---|---|
| LBS 养宠社区 | 增强活跃，但医疗内容风险高 |
| 智能摄像头实时预警 | 技术和隐私成本高 |
| 智能硬件数据接入 | 后续生态扩展 |
| 宠物保险 | 长期商业化方向 |
| 年度健康报告 | 增强用户粘性 |
| 多宠家庭协同 | 会员增强功能 |
| 行为训练模块 | 后续扩展场景 |

---

### 5.5 Won’t Have：暂不需要

| 功能模块 | 原因 |
|---|---|
| AI 直接确诊 | 合规风险高 |
| AI 自动开处方药 | 处方药必须兽医承接 |
| 自营宠物药品仓储 | 投入大、合规重 |
| 大规模开放医疗社区 | 审核和医疗风险高 |
| 实时远程诊疗直播 | 资源和责任压力大 |
| 自研智能硬件 | 供应链太重 |
| 宠物医院完整 HIS 系统 | 偏 B 端重系统，早期不做 |

## 6. 阶段性目标拆解

### MVP 阶段目标，0—3 个月

| 类型 | 目标 |
|---|---|
| 用户目标 | 完成宠物档案、AI 问诊、风险分级、提醒功能 |
| 业务目标 | 验证用户是否愿意使用 AI 问诊和线上兽医 |
| 数据目标 | 建立基础问诊漏斗和医疗安全指标 |
| 核心指标 | 档案创建率、问诊完成率、高风险转人工率、用户满意度 |

### 成长期目标，3—12 个月

| 类型 | 目标 |
|---|---|
| 用户目标 | 提高健康管理使用频率 |
| 业务目标 | 建立问诊、会员、商品、医院导流收入 |
| 数据目标 | 形成用户分层、症状分层、转化分层 |
| 核心指标 | 7 日留存、30 日留存、会员转化率、问诊付费率、复诊完成率 |

### 成熟期目标，12 个月以后

| 类型 | 目标 |
|---|---|
| 用户目标 | 成为宠物健康问题的第一入口 |
| 业务目标 | 建立平台生态和多元收入结构 |
| 数据目标 | 形成宠物健康数据资产和模型优化闭环 |
| 核心指标 | MAU、LTV、会员续费率、医生工作台使用率、医院合作转化率 |

---

## 7. 最终结论

目标拆解的核心是先让用户敢用，再让用户常用，最后让用户离不开。

V1.0 必须围绕“AI 分诊、宠物档案、风险识别、兽医承接、日常提醒”构建最小闭环；V1.1 再做医生工作台、复诊、电商、会员和慢病管理；V1.2 再扩展社区、硬件、保险和生态服务。

产品不能一开始就做“大而全”的宠物平台。合理路径是：

```text
先解决宠物异常时的判断问题
↓
再解决长期健康管理问题
↓
最后解决医疗服务和养宠生态连接问题
```
